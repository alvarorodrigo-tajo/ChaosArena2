5. Hitbox del cuerpo (caja de colision del personaje)

La hitbox se calcula en entity/Player.java metodo getHitbox() linea 257-264. El tamano depende de las variables hbWMult, hbHMult, hbOffsetX, hbOffsetY.

Valores por personaje (calibrateScale, linea 119)

PersonajehbWMulthbHMulthbOffsetXhbOffsetYscale

Goro0.35f (defecto)0.80f (defecto)0f0f4.0f
Liu Kang0.35f (defecto)0.80f (defecto)0f0f4.0f

Como ajustar la hitbox

  • hbWMult: anchura relativa al frame. Sube = caja mas ancha
  • hbHMult: altura relativa al frame. Sube = caja mas alta
  • hbOffsetX: desplazamiento horizontal en px. Positivo = derecha
  • hbOffsetY: desplazamiento vertical hacia abajo desde el tope del frame

Ver la hitbox en pantalla

Activa DEBUG_HITBOXES = true en Player.java linea 61 para ver:

  • Verde = cuerpo del personaje
  • Rojo = area de ataque extendida

6. Fisica (gravedad, fuerza del salto)

ArchivoLineaConstanteValorEfecto al cambiar

Relacion: El salto depende de ambas. Si aumentas JUMP_FORCE sin tocar GRAVITY, el personaje llegara mas alto pero tardara lo mismo en caer. Para saltos mas flotantes sube ambas con la misma proporcion.

12. Flujo completo de un golpe (resumen tecnico)

1. Boton presionado → player.attack(PUNCH) → Estado = ATTACKING → attackTime = 0
Archivo: Player.java lineas 231-239
2. Cada frame → updateAttack(delta) → attackTime avanza → ¿attackTime > hitTime?
Archivo: Player.java lineas 169-208
3. Si attackTime > hitTime → canHit(opponent) → RectF.intersects? → Si hay colision
Archivo: Player.java lineas 190-193 y 266-274
4. Calcular dano → baseDmg * multiplier → ¿critico? → opponent.takeDamage()
Archivo: Player.java lineas 194-201
5. Aplicar dano → amount * (1 - armor) → currentHealth -= reduced → Estado = HURT
Archivo: Player.java lineas 242-248
6. ¿currentHealth <= 0? → triggerResult() → Fin del combate
Archivo: Juego.java lineas 295-296
